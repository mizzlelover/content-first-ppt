# 01 · 全量摄取（Ingest）

> 目标：用最大算力**完整**消化用户提供的全部材料。**全量阅读，不是概览。**
> 原则：不得遗漏任何一个文字、标点、表格、以及**任何一张图像**。

## 硬性要求
1. **100% 覆盖**：逐文件、逐页、逐段读完。长文档分块，**每一块都要真正读**，不能只读摘要/首尾。
2. **图像必须"看"**：每张图/表/截图都要打开理解，转述其信息，并提取其中的文字、数据、图例、流程。**不能跳过图。**
3. **不改写、不评判**：摄取阶段只"接收"，不要在这一步就下结论或筛选。
4. **保留出处**：记录每条信息来自哪个文件/页/段，供后续引用与核查。

## 产物：`ingest-ledger.md`（摄取清单 = 无遗漏的可核验证据）
对每个材料来源登记：
```
### 文件：<名称/路径>（类型：doc/pdf/ppt/xlsx/img/...）
- 覆盖：[已全读 ✓]  规模：<页数/字数/表数/图数>
- 关键信息点：
  - …（逐条，带页/段定位）
- 图像清单：
  - 图1（第X页）：<内容转述 + 图内文字/数据>
- 数据/指标：<数值、单位、口径、时间>
- 疑点/待澄清：<与其他材料矛盾或表述不清之处>
```

## 大材料的处理
- 切块读取 → 每块产出要点 → 汇总；**最后自检**："是否每一节、每一张图都已登记？"
- 若材料超出一次处理量，分多轮，但**必须读完全部**再进入下一阶段。

## 放行条件
- `ingest-ledger.md` 覆盖了**所有**文件与图像；
- 没有"未读/跳过"的部分；
- 疑点已列出（进入阶段②或③处理）。

> 完成后进入 `02-structure.md`。
