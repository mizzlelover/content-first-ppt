# 02 · 客观结构化（Structure the SOURCE）

> 目标：对**原始材料本身**做客观、结构化的组织与内化。
> 关键：**这一步组织的是"原始材料"，不是 PPT。** 现在不要想页面、不要想视觉。

## 客观性三分法
把每条信息明确归类，**不许混淆、不许臆造**：
- **材料事实**：材料中明确写到的（带出处）。
- **可合理推断**：基于材料能稳妥推出的（标注"推断"）。
- **缺口/未知**：材料没有、但叙事可能需要的（标注"缺口"，留待阶段③问用户或在阶段④规避）。

## 产物：`source-model.json`（原始材料的结构化模型）
建议结构（可按材料调整，但要素要全）：
```jsonc
{
  "topic": "材料主题（中性描述）",
  "entities": [ {"name":"", "role":"", "note":""} ],          // 主体/产品/组织/角色
  "facts":    [ {"text":"", "source":"文件:页", "type":"fact|inferred"} ],
  "claims":   [ {"text":"主张", "evidence":["facts 引用"], "strength":"strong|weak"} ],
  "metrics":  [ {"name":"", "value":"", "unit":"", "basis":"口径/时间", "source":""} ],
  "timeline": [ {"time":"", "event":""} ],
  "relations":[ {"from":"", "to":"", "type":"因果/包含/对比/依赖"} ],
  "assets":   [ {"ref":"图1", "desc":"", "usable_as":"配图/数据图/流程图"} ],
  "open_questions": [ "缺口或矛盾，需澄清的问题" ],
  "glossary": [ {"term":"", "plain":"用最通用语言解释（给非专业者）"} ]
}
```

## 要点
- **先穷尽事实与数据**，再归纳主张；每个主张尽量挂上证据（facts/metrics）。
- **术语平民化**：为关键术语写一句"非专业者也能懂"的解释，供阶段④开场消除信息差用。
- **不做取舍排序**：哪些进正文、哪些进附录，是阶段④的事；这里只客观沉淀。

## 放行条件
- 事实/数据有出处；主张有证据或被标注为"弱/推断"；
- 缺口与矛盾已进入 `open_questions`。

> 完成后进入 `03-audience.md`。
