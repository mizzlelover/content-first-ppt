# 05 · 写页与渲染（Authoring & Render）

> 把 outline 落成 `deck.json`（见 `core/schema/deck.schema.json`），校验通过后渲染。

## 页面范式：问题切入 + 内容回答（默认）
- **标题 = 一个问题**（问题导向）：用受众会问的问题做大标题，引发注意、把听众聚焦到"这页怎么解决它"。
  - ✅「选址与场馆，能否支撑海关监管？」「免税/完税品类底数有多少？」
  - ❌「选址情况」（名词堆叠）、「我们的选址很好」（自夸式断言）
- **正文 = 答案**：用图/表/关键数字/极简要点**正面回答**这个问题——问题不能悬空。
- **一页一问**：一页只回答一个问题；答不下就拆页，**不要缩小字号去硬塞**。
- 例外：封面/目录/章节过渡/结尾不用问句；诉求页可用「恳请…支持哪几件事？」这类问句引出清单。

### 问题质量（决定成败 · 逐条过，这是本技能最容易翻车的地方）
问题导向 **≠ 把章节名加问号**。一个**好问题**必须同时满足：
1. **非显而易见**：答案对「**该受众**」不能是理所当然。**反例（讲给海关）**：「为何离不开海关？」——政策性免税市场当然离不开海关，等于没问，还显得低估对方。**显而易见测试**：若该受众心里会嘀咕「这还用问？」，立即作废重拟。
2. **是该受众要判断 / 会质疑 / 要拍板的问题**：从受众视角出发，问他真正关心、要决策、会挑刺的点——不是通用问题，更不是章节标题加问号。
3. **带真实张力**：问题里要含可行性 / 合规 / 风险 / 取舍的张力，正文随即正面回答。把「够不够 / 能不能」升级为「**凭什么相信**」。
4. **指向你的目标**：问题串起来要把受众一步步推向你要的决定（支持 / 批复 / 合作）。
**弱问题特征**（出现就重拟）：以「是什么 / 怎么样 / 怎么做 / 有没有」开头且答案显然；通用、不针对该受众；只是把章节名加了问号。
> 反例 → 正例（面向海关）：
> - 「这是个怎样的市场，为何离不开海关？」→「我们想和海关共建怎样的免税监管样本？」
> - 「客流能否支撑运营？」→「这个市场，凭什么能长期运营下去？」
> - 「怎么运营？」→「由谁运营？交易数据能否为海关监管所用？」

### 问题质量检查清单（12 项）
逐条自检，任一不过即重拟：
1. **受众归属**：是"谁的问题"而非"谁的主题"？能说出"这页替谁而问"。（Minto 读者问题）
2. **决策相关**：答完后受众会更新什么判断/优先级/批否？否则降为正文副标。（So-what）
3. **非显而易见**：答案对该受众不能理所当然（"显而易见测试"）。（Loewenstein 信息缺口）
4. **真实张力**：含冲突/代价/风险/约束/时间压力/机会损失。（SCQA Complication · SPIN Implication · ABT But）
5. **可回答性**：能在本页/本节被清晰回答；否则是悬念不是结构。（Minto）
6. **精确性**：具体到对象/指标/时段/边界。（Paul-Elder 苏格拉底）
7. **假设暴露**：暴露需检验的关键假设，而非把假设写成结论。（假设驱动 / Paul-Elder）
8. **价值放大**：推进到后果或收益——把"能不能/够不够"升级为"凭什么相信/否则代价是什么"。（SPIN Implication→Need-payoff）
9. **认知匹配**：要求的认知动作（理解/分析/评价/创造）与受众成熟度相称。（Bloom）
10. **心态健康**：是 Learner 不是 Judger（不预设指责，邀请共同推理）。（Marilee Adams）
11. **结构位置**：它是总问题/分支问题/证据问题？层级清楚才好排序。（issue tree）
12. **下一步牵引**：答完这页，受众自然会问下一页什么？形成问答链。（Minto Q&A）

### 按受众生成问题的流程（8 步）
1. 先定受众不定主题：写清"谁、什么场合、要做什么决定、最怕什么"。
2. 用 SCQ/ABT 写张力：共识事实 → 唯一关键冲突 → 逼出的关键问题。
3. 画问题树而非目录：顶层总问题 → MECE 分支 → 页面问题；子问题须真回答父问题。
4. 发散生成（QFT / question burst）：先批量产问题不评判，再做开放/封闭转换。
5. 用 SPIN 放大：保留已推进到 Implication/Need-payoff 的问题；少留纯 Situation。
6. 用 Bloom 设认知坡度：理解/分析 → 评价/取舍 → 方案；高成熟受众直接从分析/评价开场。
7. 苏格拉底质检：追问目的/信息/假设/视角/蕴含；经不起者删。
8. So-what 压缩：把每个标题改写到直接指向"这对该受众意味着什么"。

### 硬模板与开放/封闭
- **造句模板**（替代名词主题）：**受众角色 + 决策动作 + 当前冲突/约束 + 关键变量 + 结果含义**。
  - ✗「竞争分析」→ ✓「若对手价格战持续两季，我们哪条护城河先失效？」
  - ✗「商业模式更新」→ ✓「现金只够 X 个月时，我们为何仍应押注 Y 而非 Z？」
- **开放 vs 封闭（QFT）**：deck 前段用**开放问题**界定空间，后段用**封闭问题**逼出结论。

## 文案与措辞（大师级细节 · 必须逐条遵守）
内容是本技能的命脉。文案要做到「**放上幻灯片就是成品**」，经得起逐字推敲：
- **严禁元语言 / 旁白 / 演讲腔**：标题不出现「一句话」「综上」「接下来」「本页」「首先/其次」「我们来看」等写作旁白，也**不要用「二十分钟，回答/讲清…」这类演讲时长旁白做标题**（目录页直接用「汇报提纲」或核心问题清单）；标题与正文**直接陈述**结论或事实，不暴露写作过程。
- **封面副标题 = 定位短语（名词性）**，不是口号、更不是动词诉求。诉求类（「恳请…支持」「建议…」）只出现在**诉求页 / 结尾**，绝不上封面。
- **标题 = 一个干净的问题（问题导向）**，或必要时一句利落的断言；不加冒号套话（如「定位：」「关于：」）；用词书面、准确、得体。
- **按受众调语体**：政务 / 对外 / 领导（董事长·总经理·外部单位）等正式场合用**庄重书面**语；避免口语动词（「卡在」「破题」「走多远」「硬塞」「折腾」「省下」「补上」「配个」）、江湖气，以及拟人 / 比喻式表达（「安全大脑」「不下班」「裸奔」「护城河」——必要时降到正文，不进大标题）。**目录页与章节过渡页的标题同样遵守此语体。**
- **标签 / kicker = 简洁名词**，全篇风格统一，不加旁白式括注。
- **排比与一致**：同组要点结构对齐、字数相近；同一指标全篇格式统一（数字、单位、口径）。
- **去重**：同一数据 / 表述只在其归属页出现一次，不跨页重复。
- **数字卡只放「真·关键数字 / 极短词」**；长名词短语不要塞进巨号位。
- **不臆造**：材料中没有的数字、名称、结论一律留白或标注「待确认」，绝不编造。
- **逐页自检**：这页有无旁白？副标题是否名词定位？标题是否口语/套话？是否与他页重复？有无编造？读起来像不像正式成稿？

> 反例 → 正例：
> - 封面副标题「以免税经营为核心 · 恳请海关指导支持」→「台胞台企登陆第一家园 · 对台免税商贸新平台」
> - 标题「一句话：建一个免税经营的对台商品市场」→「免税经营是市场的核心，监管全程落在海关」

## 信息密度上限（校验器会查）
- 每页块数 ≤ 6；要点 ≤ 6 条；单要点 ≤ ~64 字（提炼成关键词，别写整段）；标题 ≤ 60 字。
- 大段文字 → 改成要点/图表/对比。
- **每页必须在版面内装得下**：`validate` 会预估块高并对可能溢出的页给出 WARN——溢出就精简或拆页，绝不缩字硬塞，也不容忍正文与页脚/分页区重叠。

## 选择块类型（`blocks[].type`）
`kpi`(大数字) · `bullets`(要点) · `compare`(对比) · `process`(流程) · `timeline`(时间线) · `table`(表) · `callout`(强调诉求) · `quote`(引用) · `two-col`(两栏) · `paragraph`(短段) · `image`(可选配图)。
- 数字/成效 → `kpi`；差异/选型 → `compare`；步骤/方案 → `process`；诉求/下一步 → `callout`。

## 页型（`kind`）
`cover`(封面) · `agenda`(目录) · `section-divider`(章节过渡) · `content`(正文) · `appendix`(附录) · `back`(结尾/诉求)。

## 面包屑 / 非线性
- 正文页设 `section`（对应 `sections` 里的 id）→ 自动出面包屑与当前节高亮。
- 需要"被问到再展开"的：正文页加 `links`，目标页 `kind:"appendix"` 且登记进 `appendix`。

## 视觉（最后一步，别喧宾夺主）
- **风格由内容选**（见 `method/06`）：用 `python core/build.py recommend --domain … --audience … --text …` 从主流库 `swiss/editorial/corporate/consulting/minimal/tech/bold` 匹配，把结果写回 `meta.style`；不是固定模板。
- **信任字阶**：不要在 deck.json 里手动改小字号。
- **配图可选**：默认不配图也应高质感。要配图时，给 `image` 块加 `gen:{prompt,...}`，并默认生成**无文字**概念图（文字用原生块叠加，避免乱码）；无图同样成立。

## 演讲者备注
- 每页写 `notes`：要点提示、被问到时跳哪页附录、口头补充。

## 渲染与迭代
```bash
python core/build.py validate work/deck.json     # 先过校验
python core/build.py html     work/deck.json -o out/   # 出单文件 HTML
```
- 有 ERROR 必修；WARN 尽量清。改 `deck.json` 后重渲染即可。
