# 06 · 视觉风格系统（内容驱动）

> 视觉是最后一步、是辅助。**先把内容/受众/叙事做对（method 00–05），再选风格。**
> 我们没有照搬任何模板，而是把**"如何定义一种风格"的方法论**抽象成一套**风格定义框架**，用它沉淀**市场主流美学预设库**，再**由内容动态选择**该用哪套。**内容为王不变**：风格在内容/受众确定之后才选，且可读性校验对所有风格强制生效。

---

## 1. 风格定义框架（一种风格 = 一组有纪律的令牌）
任何一套风格都由以下令牌定义（落在 `core/styles/<id>.json` + `core/typography.py`）：
- **配色纪律**：`bg/surface/surface_alt/text/muted/primary/accent/line` + `feature_bg/feature_text`（明暗节奏用）+ `ok/warn/bad`。强调色**单一**；文/底对比强制达标。
- **字体角色**：`sans`（正文）/ `serif`（编辑型大标题）/ `mono`（追踪标签）。每角色含拉丁+中文栈。
- **字重阶梯**：随 `look` 自动（swiss"越大越细"、editorial/clean"厚标题"、bold"重磅"）。
- **视觉寄存器 `look`**：`editorial | swiss | clean | bold` → 决定圆角、标签大小写、标题/数字字体族。
- **适配元数据 `fit`**：`domains / tones / audiences / data_density / keywords` —— 供"内容→风格"选择。

> 加新风格 = 在 `core/styles/` 丢一个含上述令牌 + `fit` 的 JSON，无需改渲染器。

## 2. 主流风格预设库（`python core/build.py styles` 查看）
| id | 调性 | 典型适合 |
|---|---|---|
| `swiss` | 瑞士国际主义：网格/无衬线/单一强调/发丝线 | 技术、工程、数据、设计、产品 |
| `editorial` | 编辑杂志：衬线标题+暖色+等宽标签 | 人文、品牌、文化、行业观察、叙事 |
| `corporate` | 商务管理层：深色克制、稳重权威 | 战略、财务、管理、集团汇报 |
| `consulting` | 咨询数据：浅色蓝灰、图表导向 | 数据分析、调研、运营报告 |
| `minimal` | 极简：近黑白+单强调、极致留白 | 通用、产品、拿不准时的安全默认 |
| `tech` | 科技产品：深色+冷色青蓝 | AI、SaaS、产品/发布、创业 |
| `bold` | 强冲击：重磅字号、高反差 | 融资路演、营销、愿景、号召 |

## 3. 内容 → 风格（动态选择，核心机制）
**流程**（在 method 03/04 拿到信息后执行）：
1. 从 `source-model` + `audience-profile` 提炼**内容信号**：
   - `domain(s)`（领域：ai/finance/data/humanities/product/…）、`tone(s)`（语气：rational/narrative/bold/…）、`audience`、`data_density`(low/mid/high)、`keywords`（或直接给 `text` 原文做关键词匹配）。
2. **推荐**：
   ```bash
   python core/build.py recommend --domain ai --domain product --tone modern --audience technical --density mid --text "<大纲或材料节选>"
   # 或： --signals signals.json
   ```
   选择器（`core/styles.py`）按 fit 打分，返回**推荐风格 + 理由 + 排名**。
3. **写回**：把选中的 id 写进 `deck.json` 的 `meta.style`，并在项目记录里留一句**选择理由**。
4. **可覆盖**：用户若指定风格，以用户为准；信号不足时回退 `minimal`。

> 经验对照（选择器实测）：AI产品+技术→`tech`；财务战略+董事会→`corporate`；人文观察→`editorial`；运维数据+部门经理→`consulting`；融资愿景→`bold`。

## 4. 通用排版纪律（所有风格共同遵守）
- **字号第一/远距离可读**：信任字阶；**宁可拆页不缩字**；校验器对正文/标题字号、对比度做 ERROR 级硬检查。
- **无动效**：平铺即呈现。
- **同类要素同字号**：同一组内的同类元素（一行 KPI 数字、同级要点、同类标签）字号**必须一致**；当需要为适配宽度而缩放时，**按整组统一缩放**（以最宽元素为准），绝不出现同组元素字号参差。渲染器已对 KPI 行做整组统一缩放。
- **结构优于装饰**：发丝线 + 顶线数据卡 + 大留白；强调色单一。
- **明暗节奏**：封面/章节过渡/结尾用 feature 反色页 + 巨号 ghost 序号。
- **面包屑常驻**：每页显示章节路径 + 当前节高亮（标杆所缺，务必保留）。
- **中文大标题分档**：≤8 字一行；2 行/9–14 字次之；≥3 行先改写文案。
- **禁用 emoji**；行内强调只点关键词。
- **配图为可选素材**：嵌入用，不画 slide 外壳；默认无字概念图、文字原生叠加；无 key 自动跳过。详见 `core/images.py`。

## 排版硬原则（逐页检讨固化 · 渲染器/校验器已自动执行）
- **P1 垂直平衡**：内容不满版时**整体居中**，消除"顶重底空"的死白（两个渲染器已对正文区做整体居中）。
- **P2 容得下不裁切**：校验器预估每页块高（**计入文字换行**，含对比卡/要点），超出即 WARN——必须精简或拆页，绝不裁切、绝不与页脚重叠。
- **P3 标题行平衡**：标题用 `text-wrap:balance`，避免末行只剩一两个字的"孤字"；写作上**标题尽量一行**。
- **P4 路标不与内容重复**：目录页 / 章节过渡页**不重复**面包屑里的章节列表；kicker 不照抄当前章节名。
- **P5 编辑级数字**：数字卡里数值与**单位/前缀分级**（单位 ≈0.5em、弱化），凸显数字本身。
- **P6 同类要素同字号**：见 §4；为适配宽度只做**整组统一缩放**。
- **P7 目录设计化**：目录用**编号（01–06）**而非平铺项目符号。
- **P8 强层级对比**：相邻层级字号要**拉开**（标题∶正文 ≈ 1.7×+，封面与巨号数字更大），别让副标题≈正文导致层级立不起来。默认字阶已加大（封面56 / 标题42 / 数字72 / 正文24 / 标签13–15）。
- **P9 极简页眉脚（留白让位内容）**：顶部面包屑贴近上边、底部操作区压到最小，上下边距收紧，把版面尽量让给内容主体。

## 量化基线（视觉设计与排版方法论）
- **正文字号**：投影路演 28pt 起（Kawasaki 30 / Duarte 28；<24pt = 文档非幻灯）。小会议室 20–24，中礼堂 24–30，大会场 32–40。
- **字阶**：模块化音阶（Bringhurst / Tim Brown），相邻级 ≥1.25×；标题∶正文 1.5–1.6；封面 ≥2×。**同页层级 ≤3 级、全 deck ≤4 级**。
- **行宽 measure**：中文正文 18–30 字/行，硬上限 40（Bringhurst 66 CPL；WCAG 1.4.8 CJK≤40；1 CJK≈2 拉丁）。
- **行距**：正文 1.35–1.5（28/40），标题 1.1–1.25（Bringhurst rhythm / USWDS）。
- **字重**：正文 400–500、小标题 600、标题 700；投影 CJK **避免 100–300**（细笔画糊边）。
- **对比度（WCAG）**：正文 ≥4.5:1、大字 ≥3:1、非文本图形/线 ≥3:1；投影关键标题/数字优先 **7:1**。
- **配色**：强调色有效占比 **5–10%**（密集图表 2–5%），>20% 失焦；不靠颜色单通道编码（叠字重/形状/标注）；60-30-10 仅为启发式。
- **网格/边距**：12 列主网格（封面/转场退 6 列）；左右边距 6–8% 页宽；**页眉脚 chrome 总高 ≤3–5% 页高**，只留必要项。
- **数据页（Tufte）**：去 3D/阴影/重边框/花纹；**直接标注**代替图例；主线宽 1.25–2.25pt；small multiples 4–12 个同尺度同序同编码；精确比较忌饼图。
- **格式塔分组顺序**：对齐轴 → 间距 → 区域底色 → 线框（先空间后装饰；组内∶组间间距 1:1.5–3）。
- **阅读动线**：稀疏叙事页用 Z、密集页用 F；首视点左上，结论/诉求收于右下；避免锯齿图文。
- **页面范式**：Assertion-Evidence（Alley，实证）——标题=完整句（我们用问题句）、视觉证据代替整页 bullet；标题带 ≈15–20%、证据区 60–75%、来源 5–8%。

## 5. 一句话
**用标杆"定义风格"的方法论，建一套主流风格预设库，让内容来挑风格——风格是微辅，内容导向是核心。**
